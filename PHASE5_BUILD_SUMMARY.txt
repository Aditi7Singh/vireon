Full Data Flow:
────────────────────────────────────────────────────────────────────
1. User opens http://localhost:3000/dashboard
2. Frontend SSR renders dashboard + fetches /api/metrics/*
3. FastAPI routes fetch from data client (Merge or ERPNext)
4. Data routed via ENVIRONMENT -> MergeAccountingClient or ERPNextClient
5. Celery beat scheduler syncs Merge.dev data every 15 minutes
6. Sync data → PostgreSQL → Anomaly detection → Alert creation
7. Frontend auto-refreshes SWR every 30 seconds
8. User clicks "Ask AI" → Chat drawer streams response via SSE
9. Agent has access to real accounting data (Merge or simulator)

Deployment Steps
────────────────────────────────────────────────────────────────────

Step 1: Get Merge.dev Credentials (5 min)
  1. Sign up at https://app.merge.dev
  2. Settings > API > Copy API key (MERGE_API_KEY)
  3. Dashboard > Link Account > Select QB Online/Xero/etc
  4. Generate account token (MERGE_ACCOUNT_TOKEN)

Step 2: Configure Environment (2 min)
  1. Copy .env.merge.example to .env
  2. Paste MERGE_API_KEY and MERGE_ACCOUNT_TOKEN
  3. Set DATA_SOURCE=merge
  4. Set other required vars (DATABASE_URL, etc)

Step 3: Verify Connection (3 min)
  python setup_merge_integration.py
  Expected: "All checks PASSED"

Step 4: Start Services (5 min)
  Terminal 1: redis-server
  Terminal 2: uvicorn backend.main:app --workers 4
  Terminal 3: celery -A tasks worker --loglevel=info
  Terminal 4: celery -A tasks beat --loglevel=info

Step 5: Test Dashboard (2 min)
  1. Open http://localhost:3000/dashboard
  2. Verify KPI cards show real QB/Xero data
  3. Try scenario: open RunwayCard → Settings → Hire 3 engineers
  4. Check chat: Ask "What's our MRR?" → Should use real data

Step 6: Deploy (varies by infrastructure)
  # Docker
  docker-compose -f docker-compose.prod.yml up -d
  
  # Heroku
  git push heroku main
  
  # Render/Railway
  Connect repo + set env vars in dashboard
