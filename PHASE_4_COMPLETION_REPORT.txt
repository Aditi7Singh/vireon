ARCHITECTURAL INTEGRATION
================================================================================

Phase 4 integrates seamlessly with existing Phase 3 stack:

LangGraph Agent (Phase 3)
  └─ get_active_alerts() tool
     └─ Calls: GET /alerts endpoint
        └─ Reads from: alerts table (populated by Phase 4)

PostgreSQL (shared)
  ├─ Phase 1: GL transactions (MariaDB/ERP)
  ├─ Phase 3: Agent sessions
  └─ Phase 4: alerts, alert_thresholds, anomaly_runs (in separate PostgreSQL via Neon)

Redis (new)
  ├─ Celery broker (task queue)
  ├─ Celery results backend
  └─ Beat scheduler state

FastAPI Backend (Phase 2, updated)
  ├─ GET /alerts (Phase 4 data)
  ├─ POST /alerts/{id}/dismiss
  ├─ /agent/chat (Phase 3)
  └─ /gl-transactions (GL endpoint for Phase 4)

Celery Workers (stateless, scalable)
  ├─ Scan for anomalies (24h)
  ├─ Cleanup old alerts (weekly)
  └─ Health checks (6h)

Agent asks "What alerts do we have?"
  → FastAPI calls get_active_alerts()
  → Fetches from alerts table (PostgreSQL)
  → Agent reasons about spikes/trends
  → Suggests owner (CFO/CTO/CEO)

   - Redis: Railway Redis (500MB free)

   - Workers: Deploy as background jobs (scale ↑)
   - Scheduler: Deploy Beat as long-running service
   
3. PRODUCTION (AWS/GCP)
   - PostgreSQL: RDS Multi-AZ
   - Redis: ElastiCache with replication
   - Celery: Kubernetes deployment with multiple workers
   - Monitoring: CloudWatch, Prometheus
   - Alerts: SNS/SQS for critical notifications

QUICK START (5 MINUTES)
================================================================================

1. Install dependencies:
   pip install -r backend/requirements.txt

2. Set environment:
   cp .env.example .env
   # Edit .env with DATABASE_URL and REDIS_URL

3. Run migrations:
   python backend/anomaly/migrations/run_migrations.py

4. Start Redis:
   redis-server  # or: docker run -d -p 6379:6379 redis:7-alpine

5. Start Celery (4 terminals):
   celery -A backend.anomaly.celery_app worker --loglevel=info
   celery -A backend.anomaly.celery_app beat --loglevel=info
   celery -A backend.anomaly.celery_app flower --port=5555
   uvicorn backend.main:app --reload --port 8000

6. Verify:
   python verify_phase4_setup.py

7. Test:
   • http://localhost:5555 (Flower dashboard)
   • http://localhost:8000/docs (FastAPI Swagger)
   • curl http://localhost:8000/alerts
