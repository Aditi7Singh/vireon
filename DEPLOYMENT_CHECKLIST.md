# Vireon Deployment Checklist

Quick reference for deploying Vireon to production on Render (backend) and Vercel (frontend).

## Pre-Deployment Checklist

- [ ] Repository is clean (no uncommitted changes)
- [ ] All tests pass locally
- [ ] `.env` file is NOT committed to git
- [ ] `render.yaml` is in repository root
- [ ] `vercel.json` is in repository root
- [ ] Both backend and frontend have been pushed to GitHub

## Render Backend Deployment

### Step 1: Prepare Repository
- [ ] Push all code to GitHub
- [ ] Verify `render.yaml` is in root directory
- [ ] Backend is in `backend/` folder
- [ ] `requirements.txt` and `requirements_agent.txt` exist

### Step 2: Create Blueprint on Render
- [ ] Visit https://dashboard.render.com
- [ ] Click "New +" → "Blueprint"
- [ ] Connect GitHub account
- [ ] Select repository
- [ ] Review services (Web, PostgreSQL, Redis)
- [ ] Click "Create Resources"

### Step 3: Wait for Initial Deployment
- [ ] Web service deploys
- [ ] Database initializes
- [ ] Redis initializes
- [ ] Check logs for any errors

### Step 4: Configure Environment Variables
In Render Dashboard → vireon-backend → Environment, set:
- [ ] `ALLOWED_ORIGINS` = (leave empty for now, will update after Vercel deployment)
- [ ] `GROQ_API_KEY` = (from https://console.groq.com)
- [ ] `SECRET_KEY` = (random secure string)
- [ ] `OPENAI_API_KEY` = (if needed)
- [ ] `MERGE_API_KEY` = (if using Merge.dev)
- [ ] `MERGE_SECRET_KEY` = (if using Merge.dev)
- [ ] `ERPNEXT_URL` = (if integrating ERPNext)
- [ ] `ERPNEXT_API_KEY` = (if integrating ERPNext)

### Step 5: Run Database Migrations
- [ ] Go to vireon-backend → Shell
- [ ] Run: `alembic upgrade head`
- [ ] (Optional) Run: `python seed_demo_data.py`
- [ ] Verify no errors in logs

### Step 6: Verify Backend
- [ ] Check https://vireon-backend.onrender.com/health/live (replace with your service name)
- [ ] Verify https://vireon-backend.onrender.com/api/v1/docs loads
- [ ] Take note of backend URL for next section

## Vercel Frontend Deployment

### Step 1: Prepare Repository
- [ ] Frontend code is in `frontend/` folder
- [ ] `vercel.json` is in root directory or `frontend/`
- [ ] Push all code to GitHub

### Step 2: Create Project on Vercel
- [ ] Visit https://vercel.com/dashboard
- [ ] Click "Add New..." → "Project"
- [ ] Select GitHub repository
- [ ] Select "Import"

### Step 3: Configure Build Settings
- [ ] Framework Preset: Next.js (auto-detected)
- [ ] Root Directory: `frontend/` (if monorepo structure)
- [ ] Build Command: `npm run build`
- [ ] Install Command: `npm install`
- [ ] Click "Deploy"

### Step 4: Set Environment Variables
After project creation, go to Settings → Environment Variables and add:
- [ ] `NEXT_PUBLIC_API_URL` = `https://vireon-backend.onrender.com` (use your backend URL)
- [ ] Click "Save"
- [ ] **Redeploy** to apply environment variables

### Step 5: Verify Frontend
- [ ] Check frontend deploys successfully
- [ ] Visit your Vercel URL
- [ ] Verify API calls work (check browser console)
- [ ] Take note of frontend URL

## Post-Deployment: Update Backend CORS

### Step 6: Update Backend ALLOWED_ORIGINS
- [ ] Go to Render Dashboard → vireon-backend → Environment
- [ ] Update `ALLOWED_ORIGINS` to: `https://your-vercel-domain.vercel.app`
- [ ] Click "Save"
- [ ] Service will auto-redeploy (check logs)

## Final Verification

- [ ] Backend health check: `/health/live` returns 200
- [ ] Backend ready check: `/health/ready` returns 200
- [ ] Frontend loads without errors
- [ ] Frontend can make API calls to backend
- [ ] Check browser console for CORS errors
- [ ] Verify database is accessible from backend
- [ ] Test key features in frontend

## Production Security Checklist

- [ ] `SECRET_KEY` is a strong random string (not hardcoded)
- [ ] All API keys are stored as environment variables
- [ ] Database password is secure (generated by Render)
- [ ] CORS is restricted to your domain
- [ ] HTTPS is enabled (auto on both Render and Vercel)
- [ ] Rate limiting is enabled on backend
- [ ] No debug mode in production (check config.py)
- [ ] Error messages don't leak sensitive info

## Troubleshooting Quick Links

- **Backend won't start**: Check `RENDER_DEPLOYMENT.md` → Troubleshooting
- **Frontend build fails**: Check `VERCEL_DEPLOYMENT.md` → Troubleshooting
- **API calls fail with CORS**: Update backend `ALLOWED_ORIGINS`
- **Database connection error**: Ensure `DATABASE_URL` environment variable is set

## Rollback Plan

If something breaks:
1. **Backend**: Go to Render → Deployments → Click previous working version → "Promote to Production"
2. **Frontend**: Go to Vercel → Deployments → Click previous working version → "Promote to Production"

## Monitoring & Logs

### Render Backend Logs
```bash
# View logs from CLI (requires Render CLI)
render logs --service vireon-backend

# Or view in Dashboard: vireon-backend → Logs
```

### Vercel Frontend Logs
- Dashboard → Deployments → Click deployment → Logs
- Or: Dashboard → Logs (real-time)

## Important Service URLs

Once deployed, you'll have:
- **Backend API**: `https://vireon-backend.onrender.com`
- **Backend Docs**: `https://vireon-backend.onrender.com/api/v1/docs`
- **Frontend**: `https://your-domain.vercel.app`
- **Postgres DB**: Internal (not publicly accessible)
- **Redis**: Internal (not publicly accessible)
